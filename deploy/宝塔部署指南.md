# CardAuth 授权管理系统 - 宝塔部署指南

## 一、环境要求

| 组件 | 版本要求 |
|------|----------|
| CentOS | 7.x / 8.x |
| 宝塔面板 | 最新版 |
| Nginx | 1.20+ |
| PHP | 8.1+ |
| MySQL | 8.0+ |

## 二、PHP环境配置

### 1. 宝塔安装PHP 8.1

软件商店 -> 搜索 "PHP 8.1" -> 安装

### 2. 安装扩展

```
fileinfo（必装）
redis（可选，用于缓存）
opcache（必装，性能优化）
```

### 3. PHP配置修改

宝塔 -> 软件商店 -> PHP 8.1 -> 设置 -> 配置文件

```ini
# 上传限制
upload_max_filesize = 50M
post_max_size = 50M

# 执行时间
max_execution_time = 300

# 禁用函数（安全加固） - 删除以下函数
disable_functions = passthru,exec,system,shell_exec,popen,proc_open,ini_alter,dl,chmod,phpinfo

# 注意：保留以下必要函数，不要禁用
# - putenv, proc_open（如果使用支付异步通知）
# - file_get_contents, curl_exec（支付接口需要）
```

## 三、项目部署步骤

### 步骤1：创建网站

1. 宝塔 -> 网站 -> 添加站点
2. 域名：填写你的域名
3. 根目录：选择 `/www/wwwroot/your-site/public`
4. PHP版本：选择 PHP 8.1
5. 数据库：创建MySQL数据库（记录数据库名、用户名、密码）

### 步骤2：上传后端代码

```bash
# 将整个 backend 目录上传到服务器
cd /www/wwwroot/your-site

# 上传后目录结构应为：
# /www/wwwroot/your-site/
#   ├── app/
#   ├── config/
#   ├── database/
#   ├── public/        <- 网站根目录
#   ├── routes/
#   ├── storage/
#   ├── vendor/
#   ├── .env
#   └── composer.json
```

### 步骤3：配置环境变量

```bash
# 复制配置文件
cp .env.example .env

# 编辑 .env
vi .env
```

修改以下配置：
```
APP_DEBUG=false                          # 生产环境关闭调试
APP_URL=http://your-domain.com

DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=card_auth                        # 你的数据库名
DB_USER=card_auth                        # 你的数据库用户
DB_PASS=your_password_here               # 你的数据库密码
DB_PREFIX=ca_

JWT_SECRET=生成一个64位随机字符串         # 使用 openssl rand -base64 48 生成

PAYMENT_TYPE=epay
PAYMENT_API_URL=https://pay.example.com
PAYMENT_APP_ID=your_app_id
PAYMENT_APP_KEY=your_app_key
```

### 步骤4：导入数据库

```bash
# 方法1：宝塔面板 -> 数据库 -> 导入
# 选择 backend/database/schema.sql 文件导入

# 方法2：命令行导入
mysql -u root -p card_auth < /www/wwwroot/your-site/database/schema.sql
```

### 步骤5：生成管理员密码哈希

```bash
# 进入项目目录
cd /www/wwwroot/your-site

# 使用PHP生成密码哈希
php -r "echo password_hash('你的密码', PASSWORD_BCRYPT, ['cost' => 12]) . PHP_EOL;"

# 复制输出的哈希值，更新数据库
mysql -u root -p card_auth -e "UPDATE ca_users SET password='粘贴哈希值' WHERE username='admin';"
```

默认管理员账号：`admin`，请在首次登录后修改密码。

### 步骤6：安装Composer依赖

```bash
cd /www/wwwroot/your-site

# 如果服务器没有composer，先安装
curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer

# 安装依赖
composer install --no-dev --optimize-autoloader
```

### 步骤7：配置Nginx

宝塔 -> 网站 -> 设置 -> 配置文件

将 `deploy/nginx.conf` 的内容替换进去，修改域名和路径。

### 步骤8：设置目录权限

```bash
# 设置storage目录可写
chmod -R 755 /www/wwwroot/your-site/storage
chown -R www:www /www/wwwroot/your-site/storage

# 设置public目录
chmod -R 755 /www/wwwroot/your-site/public
```

### 步骤9：部署前端

```bash
# 本地构建前端
cd frontend
npm install
npm run build

# 将 dist 目录内容上传到服务器 public 目录
# 或者将 dist 部署到独立的前端服务器

# 方式1：与后端同目录（public下已有index.php）
# 将 dist/* 复制到 public/ 下，注意不要覆盖 index.php 和 .htaccess

# 方式2：独立域名部署（推荐）
# 宝塔新建站点，根目录指向 dist 目录
# 在 vite.config.js 中配置正确的API地址
```

### 步骤10：配置定时任务（清理过期订单）

```bash
# 宝塔 -> 计划任务 -> 添加
# 任务类型：Shell脚本
# 任务名称：清理过期订单
# 执行周期：每5分钟
# 脚本内容：
php /www/wwwroot/your-site/public/index.php cron/clean-expired-orders
```

## 五、常见问题

### Q1: 404 Not Found
- 检查Nginx伪静态规则是否正确
- 确保 `public/.htaccess` 或 Nginx配置中 `try_files` 正确

### Q2: 数据库连接失败
- 检查 `.env` 中的数据库配置
- 确保MySQL 8.0已安装且运行
- 检查数据库用户权限

### Q3: Token验证失败
- 确保 `JWT_SECRET` 已设置且非空
- 检查PHP是否安装了 `openssl` 扩展

### Q4: 支付回调不生效
- 确保 `PAYMENT_NOTIFY_URL` 配置为公网可访问地址
- 检查防火墙是否放行对应端口
- 查看 `storage/logs` 目录下的错误日志

## 六、安全建议

1. 修改默认管理员密码
2. 定期备份数据库
3. 开启宝塔防火墙
4. 配置SSL证书（宝塔一键申请Let's Encrypt）
5. 关闭PHP错误显示（APP_DEBUG=false）
6. 定期更新系统和依赖包
